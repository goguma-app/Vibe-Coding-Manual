VS Code 필수 사용법과 GitHub 연동 공식

🧐 왜 굳이 웹사이트 대신 VS Code 프로그램을 쓸까요?

GitHub 웹사이트에서 연필 아이콘을 눌러 글을 고치는 것은 직관적이지만, 파일이 10개, 20개로 늘어나면 한계에 부딪힙니다. 내 컴퓨터(로컬)의 폴더 구조를 그대로 유지하면서 여러 문서를 동시에 드래그하고, 단축키로 미학적 검토를 수행하려면 전 세계 표준 편집기인 Visual Studio Code(VS Code)가 필수적입니다.

🛠️ Vibe 코더를 위한 VS Code 3대 핵심 단축키

어려운 코딩 기능은 다 버리고, 매뉴얼 빌드에 딱 필요한 핵심 스킬만 기억하세요.

  • 프로젝트 폴더 열기 (Ctrl + O 또는 폴더 드래그): 내가 만든 매뉴얼 작업 폴더를 VS Code 화면에 통째로 떨어뜨리면 좌측에 깔끔한 파일 내비게이터 트리가 생성됩니다.
  • 마크다운 실시간 미리보기 (Ctrl + Shift + V): 마크다운(.md) 파일을 열고 이 단축키를 누르면, 오른쪽 화면에 실제 웹사이트로 사출될 미학적 서식이 실시간 분할 창으로 출력됩니다.
  • 전체 텍스트 검색 (Ctrl + Shift + F): 프로젝트 전체 파일 중에서 특정 단어나 도메인 주소(예: goguma.app)를 한 번에 찾아내어 일괄 수정할 때 유용합니다.

🚀 내 컴퓨터와 GitHub 원클릭 연동 프로토콜

터미널에 복잡한 검은 창 명령어(git clone, push 등)를 치지 않고, 마우스 딸깍으로 내 결과물을 깃허브에 사출하는 공식입니다.

  1. GitHub 확장 연동 및 로그인 (최초 1회): VS Code 좌측 하단의 톱니바퀴(설정) 아이콘 ➡️ [Backup and Sync Settings] 또는 좌측 3번째 소스 제어 탭을 누르고 내 GitHub 계정 로그인을 연동하여 마스터 승인을 완료합니다.
  2. 스테이징 (변경 사항 담기): 내 컴퓨터에서 html이나 md 파일을 수정하면 좌측 소스 제어 탭에 알림 숫자가 뜹니다. 변경된 파일 옆의 [+ (플러스)] 아이콘을 눌러 스테이지 바구니에 담습니다.
  3. 커밋 메시지 작성: 상단 입력창에 오늘 작업한 내용을 나만의 역사로 기록합니다. (예: "feat: 4장 매뉴얼 추가")
  4. 동기화 및 밀어내기 (Commit & Push): 파란색 [Commit] 버튼을 누른 후 [Sync Changes(변경 내용 동기화)] 버튼을 누릅니다.
✅ Seamless Automation 완성 VS Code에서 [Sync] 버튼을 누르는 순간, 내 컴퓨터의 파일이 깃허브 클라우드로 안전하게 쏘아 올려지며, 연쇄 작용으로 Cloudflare Pages가 이 신호를 받아 웹사이트를 실시간 자동 갱신합니다. USB나 FTP 폴더 투척은 완전히 역사 속으로 사라집니다.
