5장. 와이드 모니터의 미학:
분할 화면 복사/붙여넣기 기술

여러 클라우드 인프라와 보안 툴을 교차 연동하다 보면, 가장 빈번하게 무너지는 구간이 바로 '세션 로그인 엉킴'과 '설정값 누락'입니다. 구글 워크스페이스 마스터 계정과 개인 클라우드플레어 계정 사이를 무의식적으로 오가다 보면 브라우저 쿠키가 꼬여 작업 흐름이 완전히 끊기게 됩니다.

바이브 코딩은 번거로운 대기 시간과 불필요한 마우스 클릭을 혐오합니다. 인프라 매핑 단계에서 멘탈을 지키고 오타 없는 완벽한 원클릭 네트워크 아키텍처를 완성하는 화면 분할(Split Screen) 동기화 전략을 구현합니다.

🖥️ 1. 복사-붙여넣기 흐름의 공간적 레이아웃

인프라 세팅 중 발생하는 휴먼 에러의 90%는 '창 전환(Alt+Tab)' 중에 발생합니다. 브라우저 탭을 전환하는 찰나의 순간에 복사한 값이 유실되거나 엉뚱한 필드에 데이터를 주입하게 됩니다. 우리의 작업 공간을 물리적으로 격리하여 인지 부하를 제로화합니다.

좌측 화면 (생산 기지)
Google Chrome (워크스페이스 프로필)

• GCP Cloud Run 엔드포인트
• Firebase RTDB URL 및 비밀키
➡️
우측 화면 (수용 기지)
Google Chrome (개인 프로필)

• Cloudflare DNS 레코드 관리 창
• SSL/TLS 암호화 모드 토글

🛡️ 2. 로그인 세션 충돌 차단: 프로필 격리 규칙

구글 클라우드(GCP)와 파이어베이스는 기업용 워크스페이스 계정(@mycompany.com)을 사용하고, 도메인 제어권을 쥔 Cloudflare는 개인 마스터 계정을 사용할 때, 단일 크롬 창에서 탭만 나눠 쓰면 구글의 다중 로그인 세션이 꼬여 "권한 없음(403 Error)" 늪에 빠지기 쉽습니다.

  • 규칙 1: 왼쪽 화면에는 반드시 워크스페이스 전용으로 독립 개설된 독립 크롬 프로필 창만 배치합니다.
  • 규칙 2: 오른쪽 화면에는 자산 관리용 Cloudflare가 로그인된 개인용 마스터 크롬 프로필 창을 배치합니다.
  • 효과: 계정 전환(Switch Account)을 위한 불필요한 로그아웃/로그인 반복 과정이 완전히 소멸됩니다.

⚡ 3. 3초 컷 인프라 매핑 실전 가이드

화면 배치가 완료되었다면 시선의 동선을 왼쪽에서 오른쪽으로만 고정하고 아래 순서로 데이터를 찰나에 전송합니다.

  1. GCP 값 추출 (좌측): CloudRun 또는 Firebase 콘솔에서 생성된 고유의 CNAME 대상 값 또는 애니캐스트 IP 주소를 Ctrl + C로 확보합니다.
  2. 시선 이동 (우측): 고개를 오른쪽으로 돌려 Cloudflare의 DNS 레코드 추가(Add Record) 필드로 즉시 마우스를 가져갑니다.
  3. 레코드 주입 (우측): Ctrl + V로 값을 붙여넣고, 프록시 상태(Proxy Status)를 설정 목적에 맞춰 주황색 구름 혹은 회색 구름으로 제어한 뒤 저장합니다.

💡 바이브 코딩 철학 노트 (Vibe Note)

와이드 모니터 분할은 단순한 UI 배치가 아닙니다. 데이터의 공급처(Source)와 수요처(Target)의 파이프라인을 물리적 공간에 그대로 일치시키는 직관적 아키텍처입니다. 양손과 시선이 규칙적으로 반응할 때 시스템 오작동률은 0%에 수렴합니다.
