시스템 구조 분석 및 트러블슈팅 공식

⚙️ 1. 최종 마스터 아키텍처 공식

[내 도메인: vibecoding.goguma.app] ➡️ [Cloudflare (보안 방패 / SSL 전체 모드)] ➡️ [Google Cloud Run (AI 웹앱 구동 엔진)] ➡️ [Firebase Realtime Database (기억 금고)]

🛡️ 2. 겪었던 핵심 장애물과 수동 돌파 공식 (Troubleshooting)

  • 404 Not Found & DB 생성 에러: AI Studio의 자동 생성 버그로 구글 표준 DB 이름 규칙(-default-rtdb)이 깨지는 현상.
    • 돌파 Vibe: 배포 전에 파이어베이스 콘솔에 먼저 진입하여 테스트 모드로 DB를 수동 선제 개설해 두면 자동 매칭되어 통과됩니다.
  • Concurrent Policy Changes (코드 10 에러): 구글 내부 스크립트의 찰나의 권한 변경 충돌 및 병목 현상.
    • 돌파 Vibe: 무한 연타를 멈추고 구글 클라우드 IAM 메뉴로 이동하여 내 마스터 계정에 직접 Firebase DB 관리자 권한을 강제 주입하여 우회합니다.
  • 525 에러 & HSTS 보안 경고: Cloudflare 방패와 구글 엔진 간의 보안 대화 방식 불일치 및 브라우저 캐시 문제.
    • 돌파 Vibe: Cloudflare SSL 설정을 '유연(Flexible)'에서 '전체(Full)'로 격상하고, 최초 등록 시에는 회색 구름(DNS Only)으로 구글 SSL 발급을 도운 뒤 나중에 주황색 구름(Proxied)을 켭니다.

🏢 3. 효율과 미학을 위한 환경 분리 원칙

  • 구글 인프라: 로그인 세션 엉킴을 막기 위해 Google Workspace 계정(@kwavemission.org) 전용 크롬 프로필을 독립 개설하여 AI Studio-Firebase-GCP를 단일화합니다.
  • Cloudflare: 자산 안전 관리 및 다중 도메인 통합을 위해 개인 마스터 계정을 유지합니다.
  • 와이드 모니터 분할 스킬: 왼쪽 화면(GCP/Firebase 값 복사) ➡️ 오른쪽 화면(Cloudflare DNS 붙여넣기) 체제로 로그인 전환 번거로움 없이 안전하게 수동 제어합니다.
  • 깃허브 조직화(Organization): 최상위 소유주를 브랜드 이름인 goguma-app으로 개설하고, 하위에 프로젝트별 저장소(vibecoding, builder 등)를 쪼개어 배포하는 테크 기업 표준 구조를 확립했습니다.

🌱 4. 바이브 코딩 3대 철학

  1. 최소 코드 (Minimal Code): 무거운 자바스크립트 소스코드 대신 명확한 구조적 프롬프트로 기능을 구현합니다.
  2. 에코/그린 코딩 (Eco-Friendly): 24시간 켜진 구형 호스팅을 버리고, 요청 올 때만 0.1초 켜지는 서버리스(Serverless)로 대기 전력과 비용을 제로화합니다.
  3. Net to Dot 데이터 청정: 거대한 개인정보 수집 그물(Net)을 치지 않고, 필요한 데이터 점(Dot)만 찰나에 찍고 사라지는 청정 웹을 지향합니다.
