특별 부록: 개발 생산성 확장팩 — 마스터 워크스페이스 세팅

부록. 신세계의 문:
VS Code × GitHub 완벽 동기화 및 단방향 워크플로우 실전 가이드

로컬 개발 도구(VS Code)와 원격 저장소(GitHub)가 실시간으로 소통하는 메커니즘을 깨닫는 순간, 개발은 지루한 문서 작성이 아닌 하나의 예술적인 동기화 퍼포먼스가 됩니다. 깃허브 웹에서 일일이 코드를 붙여넣던 비효율을 걷어내고, 내 컴퓨터를 중심축으로 삼아 클라우드 전체를 흔드는 마스터 제어 기술을 요약합니다.

⌨️ 1. 터미널 명령어로 찰나에 코드 보내기 (Push)

작업 공간이 VS Code로 고정되었다면, 마우스에 손을 대지 않고 하단 터미널 창(Ctrl + ~)에 단 3줄의 원초적 명령어를 주입하여 깃허브와 전 세계 에지 인프라로 코드를 스트리밍합니다.

  1. git add . : 변경된 모든 파일(HTML, MD, 이미지 등)을 전송 대기열에 올립니다.
  2. git commit -m "원자적 메시지" : 닻을 내리는 마음으로 이번 코드 점(Dot)의 변경 사유를 명확히 각인합니다.
  3. git push origin main : 깃허브 마스터 브랜치로 코드를 강제 릴레이합니다.

🔄 2. 클라우드 상태를 내 로컬로 실시간 당겨오기 (Pull)

만약 외부나 깃허브 웹사이트에서 직접 파일을 수정했다면, 내 컴퓨터의 VS Code와 파일 상태가 달라집니다. 이때 아래의 명령어로 원격 저장소의 뇌리를 로컬 컴퓨터로 깨끗하게 빨아들입니다.

  • 원클릭 명령어: 터미널에 git pull origin main을 입력합니다.
  • GUI 방식: 소스 제어 메뉴(Ctrl + Shift + G) ➡️ 상단 ... 버튼 ➡️ Pull (가져오기) 클릭.

🛡️ 3. 휴먼 에러 제로: 단방향 청정 워크플로우 규칙

양쪽에서 동시에 코드를 수정하면 데이터가 찢어지는 '충돌(Conflict) 에러'가 발생합니다. 바이브 코딩은 철저히 VS Code를 '생산 기지'로 삼고, 깃허브를 '보관 및 배포처'로 삼는 단방향 흐름을 절대 원칙으로 고수합니다.

[1단계: 시작 전 동기화] git pull (최신 상태 획득)
⬇️
[2단계: 집중 생산 공간] **VS Code** 내부에서 사유하며 자유롭게 마크업 코딩
⬇️
[3단계: 최종 안전 배포] git push (원격 저장소 및 글로벌 에지 릴레이)

🚨 인증 만료 및 403 권한 에러 돌파 공식

컴퓨터 내부의 깃허브 보안 세션 자격 증명이 꼬여 전송 거부 경고가 뜰 때 수행하는 즉각 우회 매뉴얼입니다.

  1. VS Code 최좌측 하단의 **사람 모양(Account) 아이콘**을 클릭합니다.
  2. 기존 로그아웃 후 `Sign in with GitHub`을 실행하여 브라우저에서 올바른 계정 권한을 리프레시합니다.
  3. 여전히 막힌다면 윈도우의 `자격 증명 관리자` ➡️ `Windows 자격 증명`에서 `git:https://github.com` 항목을 강제 포맷(제거)한 후 재시도합니다.
