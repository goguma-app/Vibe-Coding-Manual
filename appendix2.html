GitHub와 Cloudflare Pages 인증 및 실시간 연동법

🧐 연동(Integration)을 해야 수동 작업이 비로소 제로가 됩니다

내 컴퓨터에서 index.html을 생성한 뒤 Cloudflare 대시보드에 매번 마우스로 파일을 집어던져(Direct Upload) 배포하는 방식은 린(Lean)하지 못합니다. 수정할 때마다 드래그하는 피로가 누적되기 때문입니다.

Cloudflare와 깃허브 저장소를 단 한 번만 안전하게 App 인증(OAuth Handshake)으로 통일해 두면, 깃허브 저장소의 심장 박동에 맞춰 Cloudflare가 알아서 빌드와 도메인 최적화를 무한 대행해 주는 파이프라인이 개통됩니다.

🛠️ 실전 릴레이: 깃허브-클라우드플레어 무결점 연동 프로토콜

두 플랫폼의 로그인 창이 격리되지 않도록 와이드 모니터에 나란히 띄워두고 진행하면 편리합니다.

  1. Pages 생성 인터페이스 진입: Cloudflare 대시보드 왼쪽 메뉴에서 [Workers 및 Pages] ➡️ [만들기] ➡️ [Pages] 탭을 순서대로 클릭합니다.
  2. Git 연결 오토메이션 선택: '자산 직접 업로드' 대신 우측에 있는 [GitHub에 연결 (Connect to Git)] 파란색 버튼을 누릅니다.
  3. 보안 권한 위임 및 조직 전환: 깃허브 로그인 팝업창이 뜨면 소유자(Owner) 선택 화면에서 내 개인 계정이 아닌, 우리가 미리 만들어 둔 `goguma-app` 조직(Organization)을 정확하게 지정합니다.
  4. 특정 저장소 타겟팅 (보안 추천): 'All repositories(모든 저장소)' 대신 **[Only select repositories]**를 체크한 뒤, 목록에서 매뉴얼 전용 저장소인 vibecoding을 명확하게 타겟팅하여 최종 연동 승인(Install & Authorize)을 내립니다.
  5. 빌드 세팅 건너뛰기: 저장소를 선택하고 [설정 시작]을 누르면 프레임워크를 고르라는 복잡한 창이 나옵니다. 우리는 순수한 원초적 최소 코드를 지향하므로 빌드 설정을 **[None (없음)]**으로 유지하고 프레임워크 프리셋도 건너뛴 채 맨 아래 **[저장 및 배포]**를 누릅니다.
✅ 파이프라인 개통 완료 이제 Cloudflare에 https://pages.dev라는 고유한 배포 공용 주소가 무상 공급됩니다. 이 프로젝트 관리 화면에서 [사용자 지정 도메인] 탭을 눌러 vibecoding.goguma.app만 적어주면 DNS 설정까지 단 한 번에 자동 통합됩니다.

🌱 에코 코딩 관점의 실시간 연동 가치

이 연동 구조는 내 컴퓨터에서 파일을 압축하고 서버로 전송하는 데 드는 로컬 자원의 낭비를 완벽히 억제합니다. 오직 '문서 파일 내부의 글자가 변경되었을 때만' 클라우드 환경 내에서 안전하게 가상 전파 연산이 일어나므로, 가장 환경 파괴가 적고 지속 가능한 스마트 워크플로우(Smart Workflow)를 증명합니다.
